MBOX to CSV FAQ

This page collects the most common questions from customers searching for “mbox csv” support. If you do not see your issue answered below, email the operator and we will help.

What file size can I upload?

Each conversion job supports uploads up to 20 GB. The system streams directly to disk, which prevents browser memory crashes. If your archive is larger, split it into multiple files using the export tool of your email provider and queue them one after another.

Which formats are supported?

mbox-csv.com handles standard .mbox files from Gmail, Thunderbird, Apple Mail, Yahoo, Proton, Zoho, and other clients. Outlook users can upload a .pst file; the service extracts the mailbox and converts it into the same CSV layout. We can also ingest .eml collections on request.

What columns are included in the CSV?

Every row includes date, from, to, cc, bcc, subject, and message_id. These fields cover most compliance and CRM import workflows. If you need message bodies or attachments, reach out for a scoped export.

How fast is the conversion?

Processing time depends on message count. Small archives (under 1 GB) usually finish in a couple of minutes. Multi-gigabyte exports may run for longer; the progress meter on the homepage updates so you always know how many messages are done.

How is my data protected?

Uploads are stored in a private working directory tied to your job ID. When the CSV is zipped and handed back, both the source file and output archive are scheduled for deletion. HTTPS is enforced end-to-end. See the privacy policy for details.

How do I fix upload errors?

If you see a “file too large” message, the archive exceeded 20 GB. Split the export and retry. For “Not ready” download errors, the job may still be processing—wait for the status to show Ready or Done and refresh. Persistent issues? Contact support with the job ID.

Tip: bookmark mbox-csv.com so the official converter is easy to find the next time you search for “mbox csv”.
